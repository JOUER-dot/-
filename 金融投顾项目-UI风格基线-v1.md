# 金融投顾项目 UI 风格基线 v1

## 1. 项目定位

这套项目建议采用：

- 后台端：专业中台风格
- 用户端：高信任财富管理风格
- 整体关键词：稳健、专业、可信、克制、数据清晰

不建议把所有页面都做成同一种后台视觉。更合适的方式是：

- 投顾端、审核端、管理页：偏高效率、高信息密度
- 用户端、产品专区、产品详情：偏信任感、精致感、转化感

目标不是“换一套皮肤”，而是建立一套统一品牌下的双场景视觉系统。

## 2. 直接可用的风格方向

推荐采用混合策略：

- 后台框架与表格体系：参考“源力”风格
- 用户端首屏与详情页质感：参考“苹果”风格

这样组合的好处：

- 后台更利落，更适合筛选、列表、审核、状态流
- 用户端更高级，更容易建立金融产品信任感
- 品牌上仍然是一套统一的深蓝金融体系

## 4. 字体与排版

### 4.1 字体气质

建议整体保持克制，不做花哨字体切换。重点是：

- 页面标题：稳重、偏强
- 数据数字：更有力度
- 次级说明：降噪
- 标签与辅助信息：统一一套小字号系统

### 4.2 排版层级

- 一级标题：28-32px，700-800
- 二级标题：22-24px，700
- 模块标题：16-18px，700
- 正文：14px 或 15px
- 次要信息：12-13px
- 核心指标数字：28-36px，700-800

### 4.3 数字展示规则

所有净值、收益、风险、持仓占比类信息，统一遵循：

- 数字优先级高于描述
- 正负值有明确语义色
- 时间维度必须弱化但可见
- 不要让数字和说明字重一样

## 5. 圆角、边框、阴影

建议整体更偏“金融专业”，不要过软：

- 页面主卡片：16px
- 内部信息块：12px
- 按钮：10px
- Tag：999px 或 8px

阴影使用原则：

- 普通卡片优先用边框，不依赖重阴影
- 重点卡片和浮层再加轻阴影
- 整页不要到处都用渐变 + 阴影 + 大圆角

## 6. 双场景设计策略

## 6.1 后台端

适用页面：

- 工作台
- 组合产品管理
- 产品编辑
- 审核列表
- 审核详情

风格要求：

- 布局更规整
- 视觉更利落
- 操作优先级更明确
- 数据和状态表达更强

设计关键词：

- 高效
- 稳定
- 专业
- 信息密度高

## 6.2 用户端

适用页面：

- 产品专区
- 产品详情
- 我的订阅
- 登录 / 注册

风格要求：

- 首屏更有信任感
- 模块更有节奏
- 数据表达更易理解
- 强化转化路径

设计关键词：

- 值得托付
- 可信赖
- 温和但不弱
- 有金融服务感

## 7. 组件级改版规范

## 7.1 左侧导航

当前问题：

- 偏传统后台
- 品牌感弱
- 激活态不够精致

优化建议：

- 背景改为更纯净的深蓝或深灰蓝，不要太重
- 品牌区增加小型品牌标识和一句定位语
- 当前菜单项采用“浅色高亮底 + 左侧细强调条”
- 分组标题更轻，更像信息分层而不是普通标题

建议效果：

- 未选中：低对比文字
- 选中：品牌浅底 + 主色文字 + 轻微突出

## 7.2 顶栏

当前问题：

- 信息太少
- 缺少上下文感

建议增加：

- 当前页面所属模块
- 面包屑
- 当前身份标识
- 全局快捷入口或待办提示

视觉上：

- 顶栏背景更轻
- 底部分割线更细
- 用户信息区域更像“账号状态卡片”

## 7.3 页面头部

每个页面头部统一三段结构：

- 左：标题 + 一句话说明
- 中：当前关键状态或筛选结果摘要
- 右：主操作按钮

原则：

- 页面头部不要只是重复页面名
- 必须回答“这页的核心任务是什么”

## 7.4 指标卡

当前问题：

- 各页指标卡长得太像
- 指标重要性不够区分

建议拆为三类：

- 概览型指标卡：总数、净值、累计收益
- 状态型指标卡：待审、驳回、已发布
- 行动型指标卡：待处理任务、可立即操作项

表现规则：

- 数字放大
- 标签缩小
- 可选增加趋势、说明、对比值

## 7.5 状态标签

状态视觉必须统一。

建议：

- 草稿：灰
- 待审核：蓝
- 已驳回：红
- 已上架：绿
- 已下架：中灰

标签样式统一：

- 弱底色 + 中高饱和文字
- 不建议每种状态都使用高饱和纯色实底

## 7.6 筛选工具栏

当前问题：

- 快捷筛选和详细筛选分开，节奏偏碎

建议：

- 第一层：快捷筛选按钮
- 第二层：展开式高级筛选
- 查询、重置、导出、批量操作都进入统一工具带

目标：

- 让筛选成为一个完整操作区，而不是多个分散模块

## 7.7 表格

表格是后台端最关键组件。

建议：

- 行高略放大一点，提升可读性
- 表头更清晰，但不要太重
- 第一列信息支持“标题 + 标签 + 辅助说明”
- 行操作收敛，主操作直出，次级操作进入更多菜单
- 批量操作条更突出

规则：

- 表格不只是数据容器，也是任务入口

## 7.8 产品卡片

适用 `产品专区`。

建议每张卡片明确展示：

- 产品名称
- 类型
- 风险等级
- 一句话卖点
- 适合人群
- 建议持有期
- 最新净值 / 累计收益

不要只停留在“名称 + 标签 + 两块数据”。

## 7.9 图表模块

建议统一规则：

- 主曲线：品牌蓝
- 辅助曲线：浅蓝或中灰
- 正向区域：淡绿
- 风险提示：橙或红点缀
- 图表旁边必须有数据摘要

不要只给图表，没有一句“读图结论”。

## 8. 页面级改版方案

## 8.1 工作台

目标：

- 从入口页升级为角色工作台

建议：

- 投顾端：首页突出待处理草稿、待审核、近期变更、快捷创建
- 审核端：首页突出待审任务、高优先级审核、最近处理
- 用户端：首页突出我的订阅、收益变化、推荐产品

布局建议：

- 顶部：欢迎区 + 身份区 + 主任务
- 中部：核心指标 + 待办列表
- 下部：快捷入口与最近动态

## 8.2 组合产品管理

目标：

- 从普通列表页升级成任务型产品运营页

改版结构建议：

1. 顶部主工作区
2. 待处理任务面板
3. 统一筛选工具带
4. 产品列表
5. 批量操作区

重点优化：

- 让“待审核 / 驳回待改”更显眼
- 快捷筛选和复杂筛选合并
- 产品行信息突出产品状态和下一步动作
- 驳回意见与审核记录改成更完整的侧边抽屉体验

## 8.3 审核列表 / 审核详情

目标：

- 强化审核效率和判断清晰度

建议：

- 列表页突出风险、提交时间、审核优先级
- 审核详情分三栏：
  - 左：产品核心信息
  - 中：策略与配置明细
  - 右：审核动作、意见、历史记录

审核动作按钮必须强主次：

- 通过
- 驳回
- 查看历史

## 8.4 产品专区

目标：

- 从普通列表变成“有转化感的财富产品陈列页”

建议：

- 顶部做产品筛选与价值引导
- 产品卡片增加适用人群和亮点摘要
- 风险等级不只显示标签，还要体现定位
- 卡片底部要强化“查看详情”

## 8.5 产品详情

目标：

- 建立信任，推动订阅

首屏建议结构：

- 左侧：产品标题、风险等级、类型、亮点、标签
- 右侧：目标客户、建议持有期、主操作区

正文结构建议：

1. 关键指标
2. 收益表现
3. 持仓快照
4. 基金成份
5. 策略逻辑
6. 风险提示

当前页最应该优化的点：

- 首屏信息价值感不够强
- 订阅动作还不够像主任务
- 风险与收益表达没有完全拉开

## 8.6 登录 / 注册

目标：

- 强化品牌第一印象

建议：

- 左侧品牌区增加“平台价值说明”
- 不只是角色枚举，要像一套正式金融平台
- 文案更偏：
  - 智能配置
  - 风险匹配
  - 组合管理
  - 审核闭环

## 9. 数据语义规范

### 9.1 收益类

- 正收益：绿色
- 负收益：红色
- 零波动：中性灰
- 累计收益与区间收益不要同样处理

### 9.2 风险类

建议风险等级形成完整体系：

- R1：低风险
- R2：稳健
- R3：平衡
- R4：进取
- R5：高波动

展示时不只写代码，最好配一个中文语义。

### 9.3 时间类

所有时间建议统一格式和优先级：

- 主时间：更新时间 / 审核时间 / 快照日期
- 次时间：操作记录中的时间戳

时间信息不要抢主内容视觉。

## 10. 文案语气规范

整个项目的文案建议：

- 少一点系统提示腔
- 多一点专业服务感
- 后台端强调“任务”“处理”“结果”
- 用户端强调“理解”“选择”“适配”“建议”

示例：

- 不建议：暂无产品
- 建议：当前暂无符合条件的产品，可调整筛选条件后重试

- 不建议：立即订阅
- 可优化为：订阅并持续跟踪

## 11. 落地顺序

建议按这个顺序执行：

1. 先统一 `tokens.css`
2. 再统一 `theme.css`
3. 再清理 `app.css` 的全局节奏
4. 重做基础组件：
   - `SectionCard`
   - `StatCard`
   - `StatusTag`
   - 顶栏
   - 侧栏
5. 重做页面：
   - `ProductList.vue`
   - `AdvisorZoneDetail.vue`
   - `Dashboard.vue`
   - `ReviewPendingList.vue`
   - `AdvisorZoneList.vue`
6. 最后补齐登录、注册和空状态

## 12. 第一阶段可快速见效的改动

如果你想先做一版“短时间肉眼可见的升级”，优先改这 6 件事：

1. 统一颜色变量和状态色
2. 重做左侧导航激活态
3. 重做页面头部与指标卡
4. 合并筛选区，减少页面碎片
5. 强化产品详情首屏
6. 统一表格与批量操作条

## 13. 最终建议

这套金融投顾项目不要走“炫技风”，而要走：

- 后台高效
- 用户可信
- 数据清晰
- 品牌克制

最终推荐的实施方向是：

- 后台部分：采用偏“源力”的专业中台感
- 用户部分：采用偏“苹果”的财富管理质感
- 品牌统一为深蓝主色 + 金色点缀 + 清晰语义色

这会比单纯继续堆卡片和渐变，更像一个真正的金融投顾产品。
